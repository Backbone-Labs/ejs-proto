WebKit WASM Regression Repro

iOS 26.2 WebKit regression: memory.grow / large heap / Asyncify crash in WKWebView.
Reference: EmulatorJS/EmulatorJS#1143 — no ROM or third-party code required.

Test 1 — memory.grow via WASM module

Pending

Instantiates a minimal WASM module (no Emscripten, no ROM) with memory(min=2, max=32768) and calls the exported grow(1) function 4096 times to reach 256 MB. Mirrors Emscripten's ALLOW_MEMORY_GROWTH=1 path. After each grow, writes a sentinel value into the new page and reads it back to verify the grown memory is actually accessible.

Test 2 — 512 MB fixed initial allocation

Pending

Creates new WebAssembly.Memory({ initial: 8192, maximum: 8192 }) (8192 pages = 512 MB). Matches Emscripten's INITIAL_MEMORY=536870912 + ALLOW_MEMORY_GROWTH=0 fixed heap. Tests whether WKWebView allows large static WASM heaps at instantiation time.

Test 3 — WebAssembly.Memory.grow() JS API

Pending

Same incremental growth as Test 1 but driven entirely from JS via WebAssembly.Memory.grow() rather than from within a WASM module. Isolates whether the regression is in WASM execution or in the JS Memory API itself.
